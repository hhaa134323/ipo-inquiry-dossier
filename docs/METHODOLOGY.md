# IPO 问询可比案例底稿 —— 检索 + 底稿方法论

本文档是本技能唯一的方法论事实源。所有 agent 入口文件（SKILL.md / AGENTS.md / CLAUDE.md / GEMINI.md）均指向此文件，不复制正文以免漂移。

---

## 核心原则

1. **先看问询原文，再想检索方案** —— 必须从问题原文（meta.问题原文）的全部限定出发，不得用概括短标签代替。
2. **召回不做取舍，精排逐个严判** —— 两段分离，以免早期误判漏掉潜在可比案例。
3. **一切结论落到"文件+页码"** —— 严禁编造，找不到就说找不到。
4. **原文逐字引用，不经过 AI 改写。**
5. **召回只走 grep，严禁整份缓存读入上下文** —— 召回阶段只用 grep 命中行（命中页码 + 上下文 1-2 行），绝不把整份 `.txt` 缓存读进模型；需要逐字读原文的只有精排阶段。

---

## 步骤 1：准备 —— 提取 PDF 文本缓存

1. 对 `--input` 目录（默认 `./input`）下每份 PDF，调用 `scripts/extract.py`。
2. 该脚本用 pymupdf 逐页抽取文本，生成同名 `.txt` 文件，每页正文前插入一行 `[PAGE n]` 标记（n 从 1 开始）。
3. 若 `.txt` 缓存已存在则跳过，不重复抽取。
4. **绝不将整份 PDF 塞进上下文** —— 只操作缓存文件。

---

## 步骤 2：召回（高召回、机械、不取舍）

### 2.1 从问题原文生成核心检索词

以问题原文中**所有限定性成分**为起点，包括：
- 业务/产品实体（如"集成电路产品""功率器件""晶圆代工"）
- 财务/会计实体（如"毛利率""折旧摊销""期间费用率"）
- 动词/关系（如"变动趋势""差异原因""合理性"）
- 剔除/排除条件（如"剔除产能爬坡阶段影响"）

### 2.2 同义/口径扩展

对每个核心检索词，手动扩展其上位词、下位词、同义词、文件中常见替代表述。

**半导体场景示例扩展表（持续补充）：**

| 原文词 | 扩展检索词 |
|--------|-----------|
| 毛利率 | 综合毛利率、主营业务毛利率、毛利 |
| 变动 | 波动、下滑、上升、趋势、差异、变化 |
| 折旧摊销 | 折旧、摊销、固定资产折旧、无形资产摊销、折旧摊销占比、折旧率 |
| 产能爬坡 | 产能利用率、良率提升、试生产、转固、固定资产转固、在建工程转固 |
| 代工 | 晶圆代工、代工厂、foundry、委托加工、OEM |
| 集成电路 | 集成电路、IC、芯片、半导体、数字IC、模拟IC |
| 功率器件 | 功率半导体、功率器件、MOSFET、IGBT、SiC、分立器件 |
| 研发费用 | 研发投入、研发支出、研发费用率、研发投入占比 |
| 收入 | 营业收入、主营业务收入、销售收入、营收 |
| 成本 | 营业成本、主营业务成本、单位成本、制造费用、原材料成本 |

**扩展原则：** 宁可过召回也不漏。只要扩展词可能在问询回复的正文中出现，就加入搜索清单。

### 2.3 机械 grep

1. 在 `--input` 目录的 `*.txt` 中用 `grep -i` 按每个检索词逐一扫描。
2. **只记录命中页码和上下文 1-2 行**，不做判断、不做取舍。
3. 合并所有关键词的命中结果为一个**候选清单**（去重），格式如下：

```
候选清单 (YYYY-MM-DD HH:MM)

检索条件: <关键词组合>
搜索范围: <目录>

候选1: 华虹公司/8-1 2023-01-29.pdf → 命中"毛利率"+"折旧"+"功率器件" → 第125-260页
候选2: 华虹公司/8-1-2 2023-04-13.pdf → 命中"毛利率"+"12英寸"+"折旧摊销" → 第39-80页
候选3: 晶合集成/8-1 2021-08-06.pdf → 命中"毛利率"+"折旧摊销" → 第25-40页
候选4: 晶合集成/8-1 2022-03-03.pdf → 命中"毛利率"+"折旧"+"产能爬坡" → 第4-10页
...
```

4. 候选清单写入 `output/candidates.jsonl` 供精排阶段使用。

### 2.4 候选不漏原则

- 当某一关键词命中不足 3 条时，应主动做更激进的口径扩展再搜一轮。
- 同一公司的不同轮次文件、不同版本（首轮/二轮/意见落实函）全部纳入候选，不因"已有同一家"而排除。
- **召回的产出仅是候选清单，不做任何取舍判断。**

## 步骤 3：精排（逐个候选严判）

对候选清单中的每个案例，逐字阅读原文后按以下 rubric 打分。

### 3.1 精排前的准备工作

对每个候选案例，先定位并摘录：
1. **对方问询原文**：文件中"问询"段落的逐字原文（通常以"请发行人披露""请说明""请补充"等开头），记录页码。
2. **答复要点**：发行人回复中的核心陈述（管理层的答复逻辑、数据、结论），记录起止页码。
3. 摘录必须是**逐字原文**，严禁大模型改写或归纳。

### 3.2 Rubric 评分（每项 0-2 分）

#### 维度 A：同问询实质（0-2）
| 分值 | 标准 |
|------|------|
| 2 | 问询问题与目标问题实质相同——同样在问毛利率/收入/成本的变动原因、同样涉及多产品分类、同样要求分析趋势 |
| 1 | 问询问题部分重叠——主题相近但限定条件不同（如只问综合毛利率不问分产品毛利率），或仅有一半维度匹配 |
| 0 | 问询实质不同——问的是完全不同的主题（如技术路线/募投/关联交易而非财务分析） |

#### 维度 B：真问询先例，非招股书自述（0-2）
| 分值 | 标准 |
|------|------|
| 2 | 内容来自**审核问询函的回复**，包含监管机构的明确问询（有"请说明""请披露"等问询措辞），是真实的审核先例 |
| 1 | 内容来自问询回复，但引用的段落偏重发行人自述而非针对特定问题的回答（如"公司基本情况"综述性章节） |
| 0 | 内容来自招股书、募集说明书等非问询文件，或完全不是审核问询语境 |

#### 维度 C：产品/行业可比（0-2）
针对目标公司的产品结构（集成电路 IC + 功率器件）判断：
| 分值 | 标准 |
|------|------|
| 2 | 同时涉及**IC 和功率器件**两大产品线，且行业定位（晶圆代工/IDM）可比 |
| 1 | 涉及其中一大产品线（IC 或功率器件），但缺乏另一条；或行业定位略有差异（如纯设计公司 vs 代工厂） |
| 0 | 产品/行业完全不同（如封测、设备、材料），或未涉及 IC/功率器件 |

#### 维度 D：口径一致（折旧/产能爬坡处理）（0-2）
针对问题中的"剔除产能爬坡导致的折旧摊销高于可比公司因素"判断：
| 分值 | 标准 |
|------|------|
| 2 | 明确讨论了**产能爬坡→固定资产转固→折旧摊销增加→毛利率拉低**这一传导链条，并有定量分析 |
| 1 | 提到了产能爬坡或折旧摊销对毛利率的影响，但未做定量分析，或分析口径不同（如只谈折旧率不谈产能利用率） |
| 0 | 未涉及折旧摊销/产能爬坡对毛利率的影响 |

#### 维度 E：可借鉴（0-2）
| 分值 | 标准 |
|------|------|
| 2 | 回复提供了可复用的分析框架、数据模型或论证逻辑，可以直接移植或类似处理 |
| 1 | 回复有参考价值，但需要大量调整才能适配目标公司情况 |
| 0 | 回复内容过于概况或特殊（如专属于某个公司治理问题），无实际借鉴意义 |

### 3.3 阈值与取舍

- **总分 ≥ 7 分 且 各维度无 0 分** → 保留，写入 hits.jsonl
- **总分 5-6 分，或某单项维度 0 分** → 标注"弱相关，谨慎保留"，可作为备选但不作为主论据
- **总分 ≤ 4 分** → 丢弃，不写入 hits.jsonl
- 所有丢弃的候选案例需在精排报告中记录**丢弃原因**和**缺口说明**

### 3.4 缺口说明

当所有候选案例精排后**没有**合格可比案例时，须如实输出"未找到合格可比案例"，并说明：
- 哪些维度无法覆盖（如"无 IC+功率器件同时覆盖的先例"）
- 当前候选中最接近但未达标的案例及其差距（"最接近候选为 XX，但维度 D 得分为 0，因对方未涉及产能爬坡对折旧的影响"）

## 步骤 4：输出 hits.jsonl（含评分依据）

在现有格式基础上，每个命中行新增 `判可比依据` 字段：

```json
{
  "序号": 1,
  "公司": "上市公司全称",
  "轮次": "第一轮 / 第二轮 / ……",
  "日期": "YYYY-MM-DD",
  "文件": "xxx.pdf",
  "问询": {
    "页": 5,
    "形式": "文本|表格|图"
  },
  "回复": {
    "起页": 8,
    "止页": 15,
    "形式": "文本|表格|图"
  },
  "相关性": "结合问题原文的完整限定，说明该案例与本问题的可比点",
  "关键锚点": {
    "问询": ["逐字短句1（8-30字）", "逐字短句2"],
    "回复": ["逐字短句1（8-30字）", "逐字短句2"]
  },
  "判可比依据": {
    "对方问询原文": "逐字摘录对方的问询原文段落，含页码参考",
    "答复要点": "扼要摘录发行人回复的核心论点（逐字或极精简概括，保持原意）",
    "评分": [
      {"维度": "同问询实质", "得分": 2, "依据": "问的是……"},
      {"维度": "真问询先例非招股书自述", "得分": 2, "依据": "出自审核问询函回复……"},
      {"维度": "产品行业可比", "得分": 1, "依据": "涉及IC产品但未涉及功率器件……"},
      {"维度": "口径一致", "得分": 2, "依据": "明确讨论了产能爬坡→折旧摊销→毛利率……"},
      {"维度": "可借鉴", "得分": 2, "依据": "分析框架可直接移植……"}
    ],
    "总分": 9,
    "保留": true,
    "缺口说明": "功率器件维度仅部分覆盖，但IC侧分析足够详实"
  }
}
```

- 第一行仍为 **meta 对象**（与原来一致，不含判可比依据）。
- `判可比依据` 字段中的"对方问询原文"和"答复要点"必须是**逐字原文摘录**，严禁 AI 改写。
- `关键锚点` 字段用于底稿中的黄色高亮渲染。约束：
  - 每条短句 8-30 字，必须是从原文逐字截取的连续子串。
  - `问询` 侧 2-4 条，`回复` 侧 2-4 条。
  - 精排阶段模型已在逐字阅读原文来评分，附加此字段 token 成本极小，不再产生额外判断负担。
- 精排过程全记录（包括丢弃的案例）输出到 `output/ranking_report.jsonl` 备案。

---

## 步骤 5：引用纪律（硬性约束）

- 所有结论必须能落到 `文件 + 页码`。
- **严禁编造**。如果某项依据无法在缓存中找到，如实记录"未找到"。
- **严禁改写原文措辞**。引用时必须逐字一致。
- **找不到就说找不到**，不要猜测。

## 步骤 6：生成 .docx 底稿

1. 调用 `scripts/build_dossier.py`，可用 `--input` 指定 PDF/缓存目录、`--output` 指定输出目录、`--hits` 指定 hits.jsonl 路径。
2. 脚本读取 hits.jsonl 的首行作为 meta，在其中提取问题原文逐字打印于文首；其余行作为命中案例逐个处理。
3. 对于形式=`图`：用 pymupdf 将对应页渲染为 PNG 并嵌入。维持原有行为。
4. **默认（形式=`文本` / `表格`）**：自动对源页做**几何抽表 + 文本交错渲染**，不依赖 `.txt` 缓存。
   - 对每个要渲染的源页，用 PyMuPDF `page.find_tables()` 检测表格，获取每个表的 bbox 和 `extract()`（行列二维数组）。
   - 同时提取页内文本块 `page.get_text("blocks")`，排除落在表格 bbox 内的块（避免与表格内容重复），剩余的按纵向 y 坐标与表格交错排序。
   - 排版：文本块 → `clean_pdf_text()` 清理 → 正文段落；表格 → 原生 Word Table。
5. **表格三级兜底**（绝不再把表格 dump 成长段落）：
   1. **健康表**：`find_tables` 命中且满足（行数≥2 **且** 列数≥2 **且** 空单元格占比 < 50%）→ `doc.add_table(style="Table Grid")` 构建 Word 真表格，逐单元格写入 `extract()` 原始字符串（`.strip()`，不改写数字/文字）。
   2. **坏表**：检测到表格但不满足健康阈值 → `page.get_pixmap(clip=bbox, matrix=Matrix(2,2))` 截该区域为高清 PNG，`add_picture()` 嵌入。
   3. **无表**：该页未检测到表格 → 正常段落文本。
6. **锚点高亮**：对正文段落（非表格单元格），将每条 hit 中 `关键锚点.问询` 和 `关键锚点.回复` 数组内的短句在清理后的正文上做**子串匹配**。每匹配到一条，将该子串所在段落的对应位置拆分为独立 run 并施加**黄色高亮**（`WD_COLOR_INDEX.YELLOW`）。匹配规则：
   - 透过程序在清理后的整段正文内逐条 `str.find()` 实现，不涉及大模型。
   - 匹配不到或锚点为空时静默跳过，不报错、不降级、不近似改写原文去凑匹配。
   - 多条锚点可同时高亮；同一锚子串在正文中出现多次则全部高亮。
   - 表格单元格暂不参与高亮。
7. **确定性与边际成本**：上述抽表全程为 PyMuPDF / python-docx 的确定性几何算法，原文不过大模型，边际 token ≈ 0。
8. 输出到 `--output` 目录下，文件名为 `底稿_{主题}_{日期}.docx`。

## 附录 B：精排报告模板（ranking_report.jsonl）

记录所有候选的精排结果，包括保留和丢弃的：

```json
{"候选序号":1,"公司":"华虹","文件":"...","问询页":5,"总分":8,"保留":true,"判可比依据":{...}}
{"候选序号":2,"公司":"晶合","文件":"...","问询页":4,"总分":3,"保留":false,"丢弃原因":"维度A得0分——问询实质不同(问的是募投项目而非毛利率)","缺少什么":"无毛利率维度分析"}
```
